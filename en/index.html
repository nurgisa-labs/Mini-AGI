How one incoming item moves through the loop.

Demo built on fictitious data — it shows the logic of the loop, not a real conversation.

$ miniagi capture meeting.md

The incoming item enters the loop

  • Telegram, the meeting and the document collect into one stream.
  • Nothing is lost between chat, mail and notes.
$ miniagi context

Mini-AGI gathers context

  • client · project · prior decision · deadline
  • every fact carries a source or a clear “unverified” mark.
$ miniagi draft

The draft is ready

  • reply + tasks + meeting notes
  • it is a draft for a human, not an automatic send.
$ miniagi review

Risk needs approval

  • 1 risk found: an external reply to the client.
  • the external action waits for human confirmation.
  • the agent sends nothing without a human.
$ miniagi remember

The decision returns to memory

  • the confirmed decision returns to project memory.
  • the next meeting starts with ready context.

capture

./rules

Autonomy only after review.

source

risk

approval
